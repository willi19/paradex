.. py:class:: BatchRenderer(intrinsics, extrinsics, near=0.01, far=2)

   Batched nvdiffrast rasterizer that renders a mesh into every camera view.

   One instance is built per camera set; ``render`` / ``render_multi`` return
   per-serial color, soft mask, and depth (plus an instance-id map for the multi
   variant). Requires CUDA and ``nvdiffrast``. The OpenCV→OpenGL projection and a
   ``diag(1, -1, -1, 1)`` z-flip are applied internally, with a final vertical flip
   on the output to undo the GL y-origin.

   .. py:method:: render(mesh)

      Render a single mesh into every camera view.

      Dispatches on the mesh texture type (``vertex_color`` or ``triangle_uvs``);
      the obj-dict layout per type is::

          vertex_color  -> {'type', 'verts', 'faces', 'vtx_col', 'col_idx'}
          triangle_uvs  -> {'type', 'verts', 'faces', 'uvs', 'uv_idx', 'texture_tensor'}

      :param mesh: Mesh to rasterize.
      :type mesh: trimesh.Trimesh or open3d.geometry.TriangleMesh

      :returns: ``(color_dict, mask_dict, depth_dict)`` keyed by serial. ``mask`` is a
                boolean ``(H, W)`` array per camera.
      :rtype: tuple of dict



   .. py:method:: render_multi(mesh_list)

      Render multiple meshes in a single pass.

      :param mesh_list: List of Trimesh or Open3D meshes
      :param return_id: If True, return instance ID map
      :param return_torch: Return torch tensors instead of numpy

      :returns: {serial: (H, W, 3)} RGB images
                mask_dict: {serial: (H, W, 1)} soft masks
                depth_dict: {serial: (H, W, 1)} depth maps
                id_dict: {serial: (H, W, 3)} optional ID maps (1-indexed, 0=background)
      :rtype: color_dict

      .. rubric:: Example

      >>> meshes = [apple_mesh, orange_mesh, banana_mesh]
      >>> colors, masks, depths, ids = renderer.render_multi(
      ...     meshes, return_id=True
      ... )
      >>> # ids['cam_01'][y, x] = object ID at pixel (x, y)
