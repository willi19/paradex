# bodex

Grasp inference for the pringles can using grasps generated by **BODex**
(`bodex/scale010_grasp.npy`). Estimates object 6D pose from the camera rig, maps a
BODex wrist+hand grasp into robot frame, and either projects the object mesh into the
images (`object6d.py`) or visualizes the robot+object in Viser (`visualize.py`).

## Scripts
| File | Purpose |
|------|---------|
| `object6d.py` | Capture -> solve object 6D pose -> project mesh onto images, show with `cv2`, save `obj_T.npy`. |
| `visualize.py` | Same pose estimation, then load a BODex grasp, IK the wrist, and render robot + object in Viser. |
| `get_floor.py` | One-off probe: triangulate the Charuco board and print its corner positions in robot frame (floor-plane reference). Contains a `pdb.set_trace()`. |

## Usage
Run from repo root (relative paths `bodex/scale010_grasp.npy`, `marker_offset.npy`, and
`dexgraspnet/results/...` are resolved against the cwd).

```bash
# Project estimated object pose into images (visual sanity check); press any key to close
python src/inference/bodex/object6d.py

# Visualize robot grasp + object in the web Viser viewer
python src/inference/bodex/visualize.py

# Floor / charuco probe (interactive: drops into pdb)
python src/inference/bodex/get_floor.py
```

`object6d.py` opens a `cv2` window of the merged projection — press any key to dismiss.
`visualize.py` starts a Viser web server (open the printed URL).

## Inputs & Outputs
- **Reads:** local `marker_offset.npy`; object mesh `rsc/object/pringles/pringles.obj`;
  BODex grasps `bodex/scale010_grasp.npy` (`visualize.py`); current cam params + `C2R`.
- **Captures:** images to `~/shared_data/inference/grasp_eval/<timestamp>/raw`, undistorted in place.
- **Writes:** `object6d.py` -> `projected/` images and `obj_T.npy` (robot frame).
  `get_floor.py` -> raw images + cam params; prints charuco corners.
