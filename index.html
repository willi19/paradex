What is Paradex?

Paradex orchestrates a synchronized multi-camera rig and robot hardware to capture, process, and reason about dexterous manipulation. It splits into a reusable library and application scripts:

paradex/ — library

General-purpose, reusable modules: calibration, camera IO, robot control, transforms, simulation, retargeting, and visualization.

Browse the API reference →

src/ — applications

Concrete workflows that combine library modules: calibrate → capture → process → infer. Each app ships a human README and a Claude orientation file.

Architecture

Calibrate→ Capture→ Process→ Infer

Validation harnesses sanity-check any stage in isolation.

  • Distributed: capture PCs run camera daemons; the main PC orchestrates over SSH/TCP/UDP.
  • Hardware sync: UTGE900 signal generator drives synchronized multi-camera capture.
  • Robots: XArm / Franka arms; Allegro (ROS2) & Inspire (IP socket) hands.
  • Vision: ArUco/Charuco detection, COLMAP reconstruction, triangulation, undistortion.

Application guide

Detailed, per-label docs — built up one application at a time. Each label is a src/ application; the paradex/ modules it builds on are reference and link into the API docs.

Camera daemons src/camera

runs on each capture PC

Long-running daemons that expose each capture PC's cameras to the network. The main PC drives them through the *_remote.py orchestrators in src/capture/camera. Start these before any orchestrator connects, or the controller raises a ConnectionError naming the unreachable PC.

Scripts

  • server_daemon.py — ZMQ command server wrapping one CameraLoader; handles register / start / stop / heartbeat / reload / end.
  • monitor_daemon.py — Flask web dashboard of per-PC camera + controller status (port 1234).

Ports

  • 5480 — ping/pong (REP)
  • 5481 — status broadcast (PUB)
  • 5482 — commands (REP)

Run (on each capture PC)

python src/camera/server_daemon.py
python src/camera/monitor_daemon.py   # dashboard :1234

Builds on (reference)

  • paradex.io.camera_system.camera_server_daemon
  • paradex.io.camera_system.monitor_daemon
  • paradex.io.camera_system.camera_loader.CameraLoader

Quick start

Install

git clone https://github.com/willi19/paradex.git
cd paradex
pip install -e .

Calibrate (run in order)

# 1. extrinsic (main PC; press 'c' to capture, 'q' to quit)
python src/calibration/extrinsic/capture.py
python src/calibration/extrinsic/calculate.py

# 2. hand-eye (camera → robot)
python src/calibration/handeye/capture.py --arm xarm
python src/calibration/handeye/calculate.py --arm xarm

Capture (distributed)

# on each capture PC
python src/camera/server_daemon.py

# on the main PC
python src/capture/camera/image_remote.py  --save_path dataset/001
python src/capture/camera/video_remote.py  --save_path dataset/001 --sync_mode

Process & infer

python src/process/object_turntable/extract_video.py
# extract_charuco → get_rotation → extract_mask_sam3 → generate_colmap
python src/inference/grasp_w_gui/object6d.py

Exact args, keyboard controls, and main-vs-capture-PC roles are in each app's README.
