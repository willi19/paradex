Distributed robotics data pipeline

Paradex connects camera capture, robot state, calibration, and processing into one experiment pipeline.

It helps an operator collect synchronized camera and robot data, calibrate the rig, process recordings, and check pose or grasp results.

End-to-end flow

The pipeline runs in ordered stages.

Start with rig configuration and checks that do not move hardware. Move to calibration, recording, and robot-dependent checks only after configuration, transport, and camera services are working.

  1. 01 Configure

    Select the active system/current profile; verify camera serials, PC names, ports, and shared storage.

  2. 02 Validate

    Python imports, SSH launch, command/data transport, camera services, trigger timing.

  3. 03 Calibrate

    Intrinsic, extrinsic, then hand-eye when camera-to-robot geometry matters.

  4. 04 Capture

    Remote image/video capture plus robot, hand, and teleoperation streams.

  5. 05 Process

    Align timestamps, undistort, encode, reconstruct, upload, and prepare datasets.

  6. 06 Review outputs

    Pose estimates, grasp transforms, overlays, visualizers, and robot-readiness checks.

Machine roles

Which computer runs what

Paradex runs across multiple machines. The main PC coordinates work; capture PCs control camera hardware; robot control runs only where robot drivers and controllers are available.

Main PC

Runs orchestration scripts, validation entry points, dataset commands, monitoring, and aggregation.

  • src/validate/data_sender/main.py
  • src/capture/camera/*_remote.py
  • src/process/**

Capture PCs

Run FLIR/PySpin camera services and send frame/status data back.

  • src/camera/server_daemon.py
  • src/camera/monitor_daemon.py
  • camera clients launched over SSH

Robot/control host

Runs arm and hand controllers, teleoperation, kinematics, and visualization checks when hardware is connected.

  • src/capture/robot/**
  • src/validate/robot/**
  • paradex.io.robot_controller

Shared storage

Stores calibration, raw sessions, processed outputs, uploads, and reusable caches.

  • ~/shared_data
  • camera params and processed videos
  • NAS-backed processing results

Setup order

First checks to run on a fresh setup

These commands are ordered from non-motion checks to hardware-dependent checks. Run them before a long capture session.

1. Install package and select configuration

git clone https://github.com/willi19/paradex.git
cd paradex
pip install -e .

# confirm the active rig profile
ls -l system/current
cat system/current/pc.json

2. Start camera services

# on every capture PC
python src/camera/server_daemon.py

# optional status page on each capture PC
python src/camera/monitor_daemon.py

3. Validate transport and cameras

# on the main PC
python src/validate/data_sender/main.py
python src/validate/command_sender/stream_remote.py
python src/validate/camera_system/remote_camera_controller.py

4. Move to calibration and capture

python src/calibration/intrinsic/capture.py
python src/calibration/extrinsic/capture.py
# this upstream hand-eye script currently supports xArm
python src/calibration/handeye/capture.py --arm xarm

# c starts recording, s stops, q exits
python src/capture/camera/video_remote.py --save_path dataset/001 --sync_mode
